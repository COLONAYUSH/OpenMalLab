# govulncheck acceptance list for OpenMalLab.
#
# The CI vuln gate (deploy/security/vulncheck.sh) fails the build on any
# AFFECTING vulnerability govulncheck finds reachable from our Go code, EXCEPT
# the ids listed here. An id earns a place here only when it is (a) unfixed
# upstream, so there is nothing to bump to, AND (b) outside our threat model,
# with the reason written down. Every entry is dated. When upstream ships a
# fix, bump the dependency and DELETE the entry: the gate fails on anything not
# listed, and warns when a listed id is no longer found (a stale entry to drop).
#
# Reviewed: 2026-07-17.
#
# All five below live in github.com/docker/docker (the Moby SDK), which ONLY
# the trusted orchestrator imports, to spawn jailed workers against our OWN
# local daemon. Untrusted samples run in jails with no docker socket and no
# network, so they cannot reach the docker API these describe. All are unfixed
# upstream (Fixed in: N/A as of the review date). None touch an API we call:
# the orchestrator uses ContainerCreate/Start/Wait/Attach/Kill/Remove/List plus
# Ping and volume-subpath mounts only, never the cp/archive api or authz plugins.
